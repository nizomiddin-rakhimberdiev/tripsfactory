# NEVER commit a locally regenerated `importMap.js`

`src/app/(payload)/admin/importMap.js` is generated by Payload and **must stay
committed** — Vercel builds from it. But it is regenerated on every `next build`,
and it only contains the entries for plugins that are *active at build time*.

Locally there is no `BLOB_READ_WRITE_TOKEN`, so the Vercel Blob storage plugin is
inactive and the build **silently strips**
`@payloadcms/storage-vercel-blob/client#VercelBlobClientUploadHandler`.
Committing that stripped file makes the production admin at `/admin` render a
**blank white page** — the HTML and `<title>` load, nothing else does. This has
now broken production twice (fixed in `b336a19`, broken again by a sweeping
`git add -A` in `80bbe1b`).

**Rules:**
1. After any local `npm run build`, check `git status` for this file and restore
   it: `git checkout HEAD -- "src/app/(payload)/admin/importMap.js"`
2. Never `git add -A` without reading the file list first.
3. If the import map genuinely needs regenerating, do it with the token:
   `BLOB_READ_WRITE_TOKEN=... npx payload generate:importmap`
